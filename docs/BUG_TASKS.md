# バグ修正タスクリスト

修正したいバグや改善したい挙動をまとめておく。完了したら印を付け、必要に応じて関連するコミットへのリンクを残す。

## 凡例

- `[ ]` 未着手
- `[~]` 着手中
- `[x]` 完了

各項目には対象（フロントエンド `UI0` / バックエンド `barrierfree-map` / HP）と現象、期待動作を記す。

---

## フロントエンド（UI0）

### [x] 地図情報表示ONのときのローディング表示の重なり

- **対象**：`UI0/map/`
- **現象**：地図情報表示トグルをONにしたとき、「点字ブロックデータ取得中」と「OSMデータ取得中」のテキストが画面上で重なって表示される
- **期待動作**：2つのローディングテキストが重ならないように位置・タイミングを調整する
- **修正**：UI0 v1.18.7（点字ブロック側のテキストをスピナーの下に配置）

### [x] 安全確認表示が画面遷移時にも出てしまう

- **対象**：`UI0/map/`
- **現象**：安全確認の表示が、アプリ起動直後のマップ画面表示時だけでなく、設定画面 → マップ画面、ユーザー画面 → マップ画面 のような画面遷移で戻ってきたときにも出てしまう
- **期待動作**：アプリを起動した最初のマップ画面表示時のみ表示し、画面遷移で戻ってきたときには表示しない
- **修正**：UI0 v1.18.8（sessionStorageで「はい」押下を記録、同セッション内では再表示しない）

### [x] 他画面からマップ画面に戻ったときに点字ブロック・OSMデータが再読み込みされる

- **対象**：`UI0/map/`
- **現象**：プロフィール画面・設定画面など他画面からマップ画面に戻ったときに、点字ブロックデータ・OSMデータが再読み込みされてしまう
- **期待動作**：点字ブロックデータとOSMデータの読み込みは「地図情報表示ON」の操作時のみ実行する。他画面からマップ画面に戻った場合はキャッシュ済みの表示をそのまま使い、再読み込みしない
- **修正**：UI0 v1.18.10（applyMapInfoVisibilityでキャッシュがあればフェッチせずキャッシュ表示）

### [x] 他画面からマップ画面に戻ったときに地図が現在地中央へ戻ってしまう

- **対象**：`UI0/map/`
- **現象**：プロフィール画面など他画面からマップ画面に戻ったときに、地図の表示位置が現在地中央へ戻ってしまうことがある
- **期待動作**：他画面に遷移する前の地図位置・ズーム状態を維持して戻る。特に、遷移前に「現在地中央」がOFFになっていた場合は、戻った後もそのときの表示位置のまま地図を表示する
- **修正**：UI0 v1.18.9（restoreMapReturnCacheで地図情報表示の状態に関わらず位置・ズームを復元）

### [x] 上部バーの高さが画面によって違う

- **対象**：`UI0/map/`、`UI0/profile/`、`UI0/setting/`（共通の `appbar.css` 想定）
- **現象**：マップ画面・プロフィール画面・設定画面で上部バーの高さがそろっていない
- **期待動作**：3画面で上部バーの高さを統一する
- **修正**：UI0 v1.18.5（`map.css` の `.map-app-bar` 高さを 72px → 56px に変更し全画面で揃えた）

### [x] 設定画面のヘルプボタンの説明書きをプロフィール画面と揃える

- **対象**：`UI0/setting/`、`UI0/profile/`
- **現象**：設定画面のヘルプページへのボタンの説明書きと、プロフィール画面のヘルプページへのボタンの説明書きの内容（文言）が揃っていない
- **期待動作**：設定画面側のヘルプボタンの説明書きを、プロフィール画面側のヘルプボタンの説明書きに合わせる
- **修正**：UI0 v1.18.4（3言語版の設定画面の説明文を「使い方・FAQ・フォーム等」等に揃えた）

### [x] マップ画面復帰時に一瞬現在地中央が表示されてから保存位置に切り替わる

- **対象**：`UI0/map/`
- **現象**：地図情報表示をONにして地図をドラッグして表示位置を変えたあと、他画面に遷移してマップ画面に戻ると、まず現在地中央の状態で地図が表示され、数秒経ってから元々見ていた位置に切り替わる（一瞬チラつく）
- **期待動作**：マップ画面に戻った瞬間から、ドラッグ前の位置・ズームのまま表示されるようにする（現在地中央へのリセット表示を経由しない）
- **関連**：v1.18.9（`restoreMapReturnCache` の改修）で表示位置の復元自体は行うようにしたが、初期描画タイミングや地理位置取得のタイミングでまだチラつきが残っている可能性
- **修正**：UI0 v1.18.14（applyCachedLocationが同期実行段階で mapReturnCache を先に確認し、あればそちらを優先する）

### [x] 設定画面のバージョン表示のバックエンド数値を最新に合わせる

- **対象**：`UI0/version.js`（設定画面のバージョン表示で参照される）
- **現象**：設定画面に表示されるバックエンドのバージョン番号が古い値（`BACKEND_VERSION = "1.26.0"`）のまま固定されており、実際に動いているバックエンドのバージョン（現状 `barrierfree-map` v1.27.3）と一致していない
- **期待動作**：表示されるバックエンドのバージョンを、実際のバックエンドの最新バージョンに揃える
- **備考**：今後もバックエンドが更新されるたびに `UI0/version.js` の `BACKEND_VERSION` を同期する運用にする必要あり
- **修正**：UI0 v1.18.12（BACKEND_VERSIONを"1.27.3"に更新）

### [x] 地図情報表示をOFF→再度ONにしたときに地図情報が再読み込みされない

- **対象**：`UI0/map/`
- **現象**：地図情報表示をOFFにしたあと、もう一度ONにしても地図情報（点字ブロック・OSM・道情報）が再読み込みされず、前回のキャッシュがそのまま再表示される
- **期待動作**：地図情報表示を **ユーザーが明示的にONに切り替えた瞬間** は、その時点の地図表示範囲の中心位置を基準に再読み込みする。画面遷移で戻ってきた場合（ON状態のまま）はこれまで通りキャッシュを使う
- **関連**：v1.18.10 で `applyMapInfoVisibility` がキャッシュ優先になったため、トグル操作時の再フェッチパスがなくなったのが原因。トグル変更ハンドラ側でキャッシュを破棄してから `applyMapInfoVisibility` を呼ぶ、または「強制リロード」フラグを渡せるようにする等の対応が必要
- **修正**：UI0 v1.18.13（トグル変更ハンドラでON切替時に cachedXXX を破棄してから applyMapInfoVisibility）

### [x] プロフィール画像のキャッシュが効いていない

- **対象**：`UI0/profile/`、`UI0/sw.js`
- **現象**：プロフィール画面を開くたびにアイコン画像が読み直されて一瞬空白／デフォルト画像になる。v1.18.11 で導入したキャッシュ復元はあくまで `iconUrl` 文字列を localStorage に保存しているだけで、画像バイト本体は毎回ネットワークから取得し直されている可能性が高い
- **期待動作**：以前表示したプロフィール画像はオフライン or 高速にローカルから表示し、APIアクセスを待たずに即時表示する
- **方針候補**：
  - サービスワーカー（`UI0/sw.js`）で API 配下の画像レスポンス（`/uploads/*` 等）をキャッシュする
  - もしくは画像を base64 として localStorage / IndexedDB に保存し、`<img>` の `src` に直接 data URL を入れる
  - `Cache-Control` ヘッダーがバックエンド側で短すぎる場合はバックエンド側の調整も検討
- **修正**：UI0 v1.18.15（サービスワーカーが API origin の `/uploads/` 配下画像をstale-while-revalidate方式でキャッシュ）

## 改善・機能追加（バグ修正ではない）

### [x] 設定画面に公式HPへのリンクボタンを追加

- **対象**：`UI0/setting/`
- **内容**：設定画面のヘルプ画面へのボタンの**下**に、公式HPへのリンクボタンを追加する
- **リンク先**：https://kumakero-otama.github.io/StepBy_HP/
- **挙動**：押すと**新しいタブ**で公式HPが開く（`target="_blank"` + `rel="noopener noreferrer"` 想定）
- **修正**：UI0 v1.18.6（3言語版の設定画面に `.help-link.hp-link` ボタンを追加。アイコン背景を緑にして区別）

### [x] プロフィール画面のキャッシュ表示（stale-while-revalidate）

- **対象**：`UI0/profile/`
- **内容**：プロフィール画面のアイコン画像・名前などの情報をローカルにキャッシュしておき、画面表示時はまずキャッシュから即時表示する。裏側ではAPIから最新情報を取得し、取得後に画面を更新する
- **意図**：画面遷移してきたときに白い画面や読み込み中表示の時間を短くし、体感速度を上げる
- **修正**：UI0 v1.18.11（既存の `loadProfile()` のstale-while-revalidate実装に加え、3言語版のIndexのユーザー名要素直後にキャッシュ復元のインラインスクリプトを挿入し、初回ペイントから即時表示）

### [x] 設定画面の「ライト/ダーク」「文字サイズ」を実際に動作させる

- **対象**：`UI0/setting/`、UI0全体（appbar.css・各画面のCSS）
- **現状**：v1.8.10 で設定画面に「ライト/ダーク設定」「文字サイズ設定」の項目がダミーとして入ったが、変更しても実際の表示は変わらない
- **期待動作**：
  - **ライト/ダーク**：選択した値に応じて UI0 全体の配色が切り替わる（背景・前景・上部バー等）。OS設定追従モードがあれば「自動」も選べるとよい
  - **文字サイズ**：選択した値（小/中/大）に応じて UI0 全体の本文・ラベルの基本フォントサイズが変わる
- **備考**：選択値は localStorage に保存し、画面遷移後も維持。CSS変数（`--app-bg`, `--app-fg`, `--app-font-base` 等）と `:root` の上書き、または `<html>` の `data-theme="dark"` / `data-font-size="large"` 属性切替で実装するのが一般的
- **修正**：UI0 v1.18.16（最小限の実装：language_redirect.jsで早期に `<html>` の `data-theme` / `data-font-size` 属性を適用、appbar.cssにダーク配色とフォントスケール用ルールを追加、設定画面のラジオボタンで保存・即時反映。CSS適用範囲は限定的なため、画面によっては未適用のスタイルが残る可能性あり）
